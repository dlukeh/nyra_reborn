## III. Inside the Transformer Brain
**Concept Flow**
1. **Tokenization** – breaking language into numbered pieces.  
2. **Embeddings** – mapping meaning into vector space.  
3. **Self-Attention** – weighting context relevance.  
4. **Decoding** – predicting the next token given all prior tokens.  

**Analogy:**  
A symphony orchestra where each instrument (token) listens to every other before playing the next note.  

**Compare**
| Model | Core Strength | Context Style |
|--------|----------------|----------------|
| GPT-5 | Reasoning & Creativity | Orchestrator |
| Copilot | Context Analysis & Doc Synthesis | Analyst |
| Gemini 2 | Multimodal Fusion | Sensor / Researcher |

## IV. Documentation as Knowledge
### A. Human Docs
- Code comments, READMEs, and API references.  
- Written for humans → clarity, maintainability.  

### B. Model Knowledge
- Trained on billions of documented patterns.  
- Uses “latent documentation” — learned during pretraining.  

### C. Agent Integration
- Agents can **read** Markdown / docstrings at runtime via RAG (Retrieval Augmented Generation).  
- Nyra’s Memory Tiers:  
  - **STM** → reads doc excerpts.  
  - **WM** → holds snippets while reasoning.  
  - **LTM** → stores summaries she authored herself.  

**Exercise:**  
> Have Nyra explain one of her modules by reading its docstrings aloud.

## VII. Training with Copilot in VS Code / The Hub
### Vision
Copilot acts as an *in-IDE mentor*, not just an autocompleter.  

**What He Does**
- Explains unfamiliar code lines in Markdown.  
- Demonstrates doc writing & commenting style.  
- Co-authors `README.md` and design notes.  
- Walks through debugging logic verbally or via comments.  

**Workflow Example**
1. Ask: *“Copilot, explain this class in Markdown.”*  
2. Review his annotated explanation.  
3. Commit both code + explanation to repo for Nyra’s memory training.
